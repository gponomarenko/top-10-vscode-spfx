03 Step-by-Step Setup Guide for a New SPFx Project
1
Install Node.js 18 LTS (Required)
SPFx v1.20+ requires Node.js 18 LTS. Use a version manager (nvm or nvs) to switch Node versions per project — essential when you maintain multiple SPFx solutions with different framework requirements.
# Install nvm (Windows) winget install CoreyButler.NVMforWindows # Install nvm (macOS/Linux) curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash # Install and use Node 18 LTS nvm install 18 nvm use 18 node --version # Should output v18.x.x
⚠ Version Matters: SPFx 1.20+ → Node 18 LTS. SPFx 1.17–1.19 → Node 16 LTS. Never use odd-numbered Node versions for SPFx.
2
Install Global SPFx Prerequisites
Install Gulp CLI and Yeoman globally, then the SharePoint Yeoman generator. These are the scaffolding and build toolchain foundations for every SPFx project.
# Install global toolchain npm install -g gulp-cli yo # Install SharePoint Yeoman generator (latest SPFx) npm install -g @microsoft/generator-sharepoint # Verify installations yo --version # Should output 4.x+ gulp --version # Should output CLI version 2.x+
💡 SPFx Toolkit shortcut: After installing the SPFx Toolkit extension, use the Validate local setup button in the sidebar. It checks all dependencies and installs missing ones for you.
3
Install Core VS Code Extensions
Install the essential SPFx extensions. The fastest approach is installing the SPFx Essentials pack (which bundles #07–#09 and the Rencore tools), then adding SPFx Toolkit and SharePoint Embedded separately.
# Option A: Install the full essentials pack (recommended for new developers) code --install-extension eliostruyf.spfx-essentials # Option B: Install extensions individually code --install-extension m365pnp.viva-connections-toolkit code --install-extension eliostruyf.spfx-snippets code --install-extension eliostruyf.vscode-spfx-localization code --install-extension eliostruyf.vscode-spfx-task-runner code --install-extension RencoreGmbH.vscode-spfx-deploy-package code --install-extension RencoreGmbH.rencore-tenant-wide-spfx-extension-deployment # For SharePoint Embedded development code --install-extension SharepointEmbedded.ms-sharepoint-embedded-vscode-extension # Optional: classic SharePoint / hybrid workflows code --install-extension SiteGo.spgo
💡 Workspace recommendation: Add these to a .vscode/extensions.json file in your repo so teammates get prompted to install them automatically.
4
Add Recommended Extensions to Your Repository
Create a .vscode/extensions.json file to ensure all teammates get prompted to install the same extensions when they open the project in VS Code.
// .vscode/extensions.json { "recommendations": [ "m365pnp.viva-connections-toolkit", "eliostruyf.spfx-snippets", "eliostruyf.vscode-spfx-localization", "eliostruyf.vscode-spfx-task-runner", "RencoreGmbH.vscode-spfx-deploy-package", "RencoreGmbH.rencore-tenant-wide-spfx-extension-deployment", "rencore.vscode-spfx-script-checker", "dbaeumer.vscode-eslint", "esbenp.prettier-vscode" ] }
5
Scaffold a New SPFx Project
Create a new project using the SPFx Toolkit guided form (recommended) or the Yeoman generator directly. The Toolkit also lets you clone from the PnP sample gallery with one click.
# Via Yeoman (terminal) — interactive Q&A mkdir my-spfx-webpart && cd my-spfx-webpart yo @microsoft/sharepoint # Recommended answers for a React web part: # ? What is your solution name? → my-spfx-webpart # ? Which type of client-side component to create? → WebPart # ? What is your Web part name? → HelloWorld # ? Which template would you like to use? → React # Install dependencies after scaffolding npm install
💡 SPFx Toolkit alternative: Open the SPFx Toolkit sidebar → click Create a new project → fill in the guided form. You can also add PnPjs, Reusable Controls, and NVM config in a single step from the form.
6
Configure the Local Workbench & Start Developing
Run the local development server to test your web part in the SharePoint Workbench. Use the SPFx Task Runner sidebar for one-click Gulp task execution, or run commands directly.
# Start local development server + open SharePoint Workbench gulp serve # For SPFx Extensions (must use live SharePoint page) # Edit config/serve.json first, then: gulp serve --nobrowser # Build for production gulp bundle --ship gulp package-solution --ship # Clean build artifacts gulp clean
⚠ Extensions Note: Application Customizers, Header/Footer, and ListView Command Sets cannot use the local Workbench. Set the pageUrl in config/serve.json to a real SharePoint page URL.
7
Deploy to SharePoint with Rencore Deploy SPFx Package
After running the production build, deploy the .sppkg package to SharePoint without leaving VS Code using the Rencore Deploy extension.
# After building for production, the .sppkg is here: ./sharepoint/solution/my-spfx-webpart.sppkg # To deploy via Rencore extension: # 1. Right-click the .sppkg file in VS Code Explorer # 2. Select "Deploy in the app catalog" from context menu # 3. Enter your SharePoint tenant URL when prompted # 4. Authenticate via OAuth in the browser window # 5. Done! Package is uploaded and deployed # For tenant-wide deployment: Alt+right-click → "Deploy to all SharePoint sites" # To persist your tenant URL (skip the prompt next time): # Add to .vscode/settings.json: { "rencoreSpfxDeploy.sharePointUrl": "https://yourtenant.sharepoint.com" }
8
Set Up CI/CD Pipeline (Optional but Recommended)
Use SPFx Toolkit to generate a GitHub Actions or Azure DevOps pipeline with automated build and deploy steps. The extension creates the YAML file and guides you through the required app registration setup.
# Via SPFx Toolkit UI: # Open SPFx Toolkit sidebar → Actions panel # Click "Create CI/CD workflow" → choose GitHub Actions or Azure DevOps # Follow the guided Entra app registration steps # Generated GitHub Actions workflow (.github/workflows/deploy.yml) # — builds, packages, and deploys to SPO on push to main # For Azure DevOps, the toolkit generates: # azure-pipelines.yml with the correct node version matrix # and app registration with certificate-based auth
💡 AI-assisted: In GitHub Copilot Chat, type @spfx /code to get context-aware help with your SPFx component code, or @spfx /setup for guided environment troubleshooting from the SPFx Toolkit chat participant.
9
Add Localization Support
SPFx projects include localization files by default. Use the SPFx Localization extension to manage string entries with inline previews and navigate between usage and definition seamlessly.
# Localization files are in: src/webparts/<name>/loc/ # en-us.js ← English strings (default) # mystrings.d.ts ← TypeScript interface # Usage in component (hover over key to see value inline): import * as strings from 'HelloWorldWebPartStrings'; // strings.PropertyPaneDescription // strings.BasicGroupName # Add a new locale file for French: # Create src/webparts/helloWorld/loc/fr-fr.js # SPFx Localization ext will highlight missing keys automatically
10
Manage & Upgrade Existing Projects
For existing SPFx projects, use SPFx Toolkit's upgrade and validation actions to keep solutions current with the latest SPFx framework versions without manual dependency hunting.
# Validate your project's dependencies (SPFx Toolkit sidebar → Actions): # "Validate project" → lists all outdated dependencies # Upgrade SPFx version using CLI for Microsoft 365 (under the hood): # SPFx Toolkit sidebar → Actions → "Upgrade project" # Generates a migration report with exact npm commands to run # Or manually via CLI for Microsoft 365: npm install -g @pnp/cli-microsoft365 m365 spfx project upgrade --output md > upgrade-report.md
